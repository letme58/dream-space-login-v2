# 到梦空间登录加密分析

## 1. 加密流程分析

### 1.1 核心加密流程

1. **参数准备**：构建登录请求参数（手机号、密码等）
2. **添加时间戳**：添加 `j0` 参数，值为当前时间戳
3. **添加签名**：添加 `i0` 参数，值为签名值
4. **参数排序**：使用 TreeMap 对参数进行排序
5. **生成随机密钥**：生成 16 位随机密钥
6. **RSA 加密**：使用 RSA 公钥加密随机密钥
7. **AES 加密**：使用随机密钥和固定 IV 加密排序后的参数
8. **拼接数据**：拼接加密数据和加密后的随机密钥
9. **Base64 编码**：对拼接后的数据进行 Base64 编码，生成最终的 `d` 参数

## 3. 获取密钥方法

### 3.1 Frida Hook
1. **安装 Frida**：`pip install frida-tools`
2. **编写 Hook 脚本**：
   ```javascript
   Java.perform(function() {
       var NativeUtil = Java.use("com.jingcai.apps.qualitydev.utils.NativeUtil");
       
       NativeUtil.getNetSignModuleKey.implementation = function(context) {
           var result = this.getNetSignModuleKey(context);
           console.log("RSA 公钥模块: " + result);
           return result;
       };
       
       NativeUtil.getNetSignExponent.implementation = function(context) {
           var result = this.getNetSignExponent(context);
           console.log("RSA 公钥指数: " + result);
           return result;
       };
       
       NativeUtil.getNetSignReIV.implementation = function(context) {
           var result = this.getNetSignReIV(context);
           console.log("AES 加密 IV: " + result);
           return result;
       };
   });
   ```
3. **运行 Hook**：`frida -U -f com.jingcai.apps.qualitydev -l hook.js --no-pause`

### 3.2 反编译 Native 库
1. **提取 APK**：使用 APK 工具提取应用
2. **获取 Native 库**：从 `lib` 目录获取 `libappc-cv-lib.so`
3. **反编译 Native 库**：使用 IDA Pro 或 Ghidra 等工具
4. **查找硬编码值**：搜索字符串或常量

### 3.3 从网络请求中提取
1. **抓包**：使用 Charles 或 Fiddler 抓取登录请求
2. **分析参数 d**：尝试逆向破解参数 d 的结构
3. **暴力破解**：尝试常见的 AES IV 值

## 4. 注意事项

- 应用可能会定期更新加密密钥
- Native 库可能使用了混淆或加密技术
- 实际使用时需要确保所有参数与应用保持一致
- 签名算法的实现可能比较复杂，需要仔细分析

## 5. 结论

到梦空间应用的登录加密采用了混合加密方案：
1. 使用 RSA 加密随机生成的 AES 密钥
2. 使用 AES-CBC 加密实际的登录参数
3. 密钥存储在 Native 库中，增加了逆向难度

要成功模拟登录，需要获取真实的 RSA 公钥和 AES IV 值，或者通过 Hook 等方法绕过加密验证。
